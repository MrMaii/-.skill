# 04 汇报沟通方法论

> 状态：草稿（待对齐） ｜ 最后更新：初始化

本篇梳理「向上汇报体」所依据的成熟沟通方法论，作为风格规则（09 号）的理论骨架。

## 1. 核心方法论一览

| 方法 | 来源 | 一句话 | 在本项目的作用 |
| --- | --- | --- | --- |
| BLUF | 军事/商务沟通 | 结论先行 | 输出第一句给结论 |
| 金字塔原理 | 巴巴拉·明托（麦肯锡） | 结论先行、归类分组、逻辑递进 | 信息分层骨架 |
| SCQA | 明托 | 情境-冲突-问题-答案 | 复杂问题的引入结构 |
| PREP | 演讲/表达 | 观点-理由-例证-重申 | 单点论证结构 |
| STAR | 行为面试 | 情境-任务-行动-结果 | 复盘/成果汇报 |
| 5W2H | 管理 | 何事/为何/何人/何时/何地/如何/多少 | 计划与方案完整性 |
| 闭环（PDCA + 反馈） | 管理 | 凡事有回音 | 任务汇报的完整性 |

## 2. BLUF：结论先行

**Bottom Line Up Front**——把最重要的结论/建议放在最前面。

- 适用：几乎所有汇报输出的开头。
- 模板：「结论：……（一句话）。理由有三点：……」
- 反例：用三段背景铺垫后才说出答案。

## 3. 金字塔原理（核心骨架）

四条基本规则：
1. **结论先行**：先说中心思想；
2. **以上统下**：上层是下层的概括；
3. **归类分组**：同层要点属于同一逻辑范畴；
4. **逻辑递进**：同层按时间/结构/重要性等顺序排列。

> 本 Skill 的「结论 → 要点 → 细节」分层即金字塔结构的落地。

```
        中心结论
       /    |    \
   要点1  要点2  要点3
   /  \    |     /  \
 细节  细节 细节  细节 细节
```

## 4. SCQA：复杂问题的开场

- **S 情境（Situation）**：双方共识的背景；
- **C 冲突（Complication）**：出现了什么变化/问题；
- **Q 问题（Question）**：因此要回答什么；
- **A 答案（Answer）**：我的结论/建议。

适用：需要让领导先「进入语境」的较复杂汇报。日常简短汇报可直接 BLUF。

## 5. PREP：单个观点的论证

- **P 观点（Point）** → **R 理由（Reason）** → **E 例证（Example）** → **P 重申（Point）**

用于支撑「为什么我建议 A 方案」这类局部论证。

## 6. STAR：成果与复盘汇报

- **S 情境 / T 任务 / A 行动 / R 结果**

用于汇报「我做了什么、效果如何」，强调结果与可量化产出。

## 7. 5W2H：方案完整性检查

提交方案/计划时自检：What / Why / Who / When / Where / How / How much。
确保领导不需要追问就能掌握全貌。

## 8. 闭环模型（Closed Loop）

中文职场名言：**凡事有交代，件件有着落，事事有回音。**

闭环四要素（本项目采用）：
1. **进展**：现在到哪一步了；
2. **风险/阻塞**：有什么问题、影响多大；
3. **所需支持**：需要领导给什么资源/决策；
4. **下一步**：接下来做什么、何时反馈。

## 9. 方法论的组合使用（决策树）

```
是简短日常汇报？
  ├─ 是 → BLUF（结论 + 1~3 要点）
  └─ 否 → 需要铺垫语境？
            ├─ 是 → SCQA 开场 + 金字塔展开
            └─ 否 → 金字塔（结论 → 要点 → 细节）
是任务/进展类？ → 叠加 闭环四要素
是成果复盘类？ → 用 STAR
有决策点？ → 选项 + 推荐（PREP 论证推荐项）
是提交方案？ → 用 5W2H 自检完整性
```

## 10. 与心理学的对应（呼应 03 号）

- BLUF/金字塔 → 降低认知负荷、首因效应；
- 选项+推荐 → 选择架构、决策效率；
- 闭环（含风险） → 风险可见、信任的「可靠」维度；
- 得体语气 → 被尊重感、信任的「善意」维度。
