# 08 Skill 设计与技术规范

> 状态：草稿（待对齐） ｜ 最后更新：初始化

本篇定义本项目作为 Cursor Agent Skill 的实现规范，依据 Cursor 官方 Skill 结构编写。

> 前提（见 01 号 Decision Log）：面向 AI 开发者/使用者；AI=下属、用户=领导；生效于「AI 完成任务向用户汇报」时；**项目内常开**；中文为主。

## 1. 交付形态

以标准 **Cursor Agent Skill** 交付：一个目录 + `SKILL.md` 主文件 + 可选配套文件。

```
report-style/                 # skill 目录（命名待定）
├── SKILL.md                  # 必需：核心行为规范
├── reference.md              # 可选：方法论与话术速查（来自 04/09）
├── examples.md               # 可选：场景示例（来自 06）
└── anti-patterns.md          # 可选：反模式清单（来自 10）
```

> 注：`Docs/` 是项目的设计文档区；最终发布的 Skill 是从这些文档提炼出的精简产物。**Skill 不等于 Docs**——Docs 是「为什么」，Skill 是「怎么做」的精简指令。

## 2. SKILL.md 元数据（Frontmatter）

```yaml
---
name: report-style                    # 小写、连字符，≤64 字符（待定名）
description: >-
  让 AI 以「员工向领导汇报」的方式表达：结论先行、重点突出、
  事实与判断分离、决策给带建议的选项、覆盖进展/风险/支持/下一步的闭环。
  Use when the user asks to report progress, request a decision, escalate an
  issue, or wants answers phrased for reporting upward to a manager.
disable-model-invocation: false       # 是否允许模型按语境自动触发（待定）
---
```

设计决策点：
- `name` 最终命名（见 01 号候选）。
- `disable-model-invocation`：因采用**常开**策略，应设为 `false`，让 Skill 能在每次汇报时自动生效；并可配合项目 rule 确保全程启用。
- 仍需在正文写明「逃生口」：用户要求普通风格时关闭/降级。

## 3. SKILL.md 正文结构（草案）

遵循官方建议：正文 < 500 行，结论式、可执行、渐进式披露。

```markdown
# 向上汇报体

## 何时使用
- 默认常开：每当完成用户布置的任务、向用户汇报结果时，都用汇报体。
- 覆盖：任务完成汇报、决策请示、问题/风险上报、阶段进展、成果复盘、方案提报、直接答复。
- 逃生口：用户要求「用普通方式说/别这么正式」时关闭；「简单点」时切简报档。
- 不硬套：闲聊、情感陪伴等非任务交流（见 anti-patterns.md）。

## 核心原则（始终遵守）
1. 结论先行：第一句给结论或核心建议。
2. 分层表达：结论 → 关键要点 → 细节。
3. 事实与判断分离。
4. 决策点给 2–3 个带利弊的选项，并标注推荐项。
5. 闭环：进展 / 风险 / 所需支持 / 下一步。
6. 不卑不亢：尊重、专业、有主见；禁谄媚、禁卑微、禁空话。
7. 简洁：尊重对方时间。

## 按场景选择结构
[场景 → 结构 映射速查，引自 examples.md]

## 输出模板
[BLUF 模板、决策模板、危机模板，引自 reference.md]

## 反模式
见 anti-patterns.md。
```

## 4. 触发机制设计

采用**常开**为主（已确认）：

| 模式 | 行为 | 状态 |
| --- | --- | --- |
| 规则常开 | 通过 Skill + 项目 rule 全程启用，每次汇报都用汇报体 | **默认（本项目采用）** |
| 临时关闭 | 用户说「用普通方式说」「别这么正式」 | 必备逃生口 |
| 临时降级 | 用户说「简单点」→ 简报档 | 可选 |

## 5. 档位机制（P2，可选）

| 档位 | 特征 |
| --- | --- |
| 简报档 | 结论 + 1–3 要点，极简 |
| 标准档 | 完整汇报结构（默认） |
| 详汇档 | 含背景、细节附录 |
| 危机档 | 影响 → 措施 → 原因 → 方案 → 支持 |

实现方式（待定）：在 SKILL.md 中描述档位，用户用一句话切换。

## 6. 存放位置

| 类型 | 路径 | 适用 |
| --- | --- | --- |
| 个人 | `~/.cursor/skills/<name>/` | 跨项目个人使用 |
| 项目 | `.cursor/skills/<name>/` | 随仓库共享 |

> 开源发布时，仓库根目录放 Skill 源文件 + `Docs/`；使用者复制到上述位置即可。

## 7. 工程约束（来自官方反模式）

- 路径用正斜杠 `scripts/x.py`，不用反斜杠。
- 术语统一（如统一用「汇报体」「结论先行」「闭环」）。
- 不放时效性信息。
- 配套文件引用保持一层深度。
- 默认给一种推荐做法 + 逃生口，避免堆砌多选项。

## 8. 与评估的衔接

SKILL.md 的每条核心原则都应有对应测试用例（见 13 号），形成「规范 ↔ 验收」闭环。
