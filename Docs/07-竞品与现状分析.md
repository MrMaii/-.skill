# 07 竞品与现状分析

> 状态：草稿（待对齐） ｜ 最后更新：初始化

## 1. 分析目的

了解「让 AI 用某种风格说话」这一品类的现状，明确本项目的差异化定位。

## 2. 现状概览

目前让 AI 改变表达风格主要有几类做法：

| 类别 | 代表形式 | 特点 | 局限 |
| --- | --- | --- | --- |
| 通用系统提示词 | 「请专业地回答」 | 简单、即用 | 笼统、不稳定、无方法论 |
| 角色扮演 Prompt | 「你是一位资深顾问」 | 有人设 | 偏人设，缺结构化汇报规范 |
| 风格类 Prompt 合集 | 各类 Prompt 仓库 | 数量多 | 零散、缺乏体系与评估 |
| 写作框架工具 | 金字塔/SCQA 模板 | 有方法论 | 偏写作模板，非 AI 行为 Skill |
| Agent Skill | 结构化 Skill 文件 | 可复用、可版本化 | 专注「向上汇报」的较少 |

## 3. 差异化定位

本项目的独特性：

1. **专注一种高价值范式**：不是泛泛「专业」，而是聚焦「员工向领导汇报」这一被验证有效的沟通模式。
2. **方法论 + 心理学双支撑**：有金字塔/BLUF/闭环等方法论，也有接收方心理学依据，而非拍脑袋。
3. **Skill 形态、可工程化**：以标准 Skill 交付，可触发、可调档、可评估、可版本化。
4. **开源、可改造**：提供清晰结构与测试用例，社区可复用为「风格 Skill」范本。
5. **有明确边界与反模式**：明确「不该谄媚、不该滥用」，而非无脑套用。

## 4. 竞品对比（维度打分 · 定性）

| 维度 | 通用提示词 | 角色扮演 | Prompt 合集 | 本项目 |
| --- | --- | --- | --- | --- |
| 结构化程度 | 低 | 中 | 低 | 高 |
| 方法论支撑 | 无 | 弱 | 弱 | 强 |
| 可复用/版本化 | 弱 | 弱 | 中 | 强 |
| 可评估 | 无 | 无 | 弱 | 有（测试用例） |
| 边界与反模式 | 无 | 弱 | 弱 | 明确 |
| 开源协作 | — | — | 部分 | 是 |

## 5. 风险与挑战

- **同质化风险**：「专业回答」类提示词众多，需用方法论 + 评估建立壁垒。
- **效果可感知性**：风格改变需要让用户「明显感到不同且更好」。
- **滥用风险**：被误用为谄媚或形式主义，需靠反模式约束。

## 6. 机会点

- 职场 AI 使用快速增长，「能直接拿去汇报」是高频刚需。
- 缺少体系化、可评估、开源的「汇报体」Skill，存在空位。
- 可延展为「沟通风格 Skill 家族」（如对外谈判体、客服体等）。
