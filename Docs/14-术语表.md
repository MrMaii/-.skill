# 14 术语表

> 状态：草稿（待对齐） ｜ 最后更新：初始化

统一项目核心概念的定义，确保各文档术语一致。

| 术语 | 定义 |
| --- | --- |
| 汇报体 | 本项目的核心风格：以「员工向领导汇报」的方式表达，结论先行、结构化、得体专业。 |
| 向上沟通 / 向上管理 | 下属向上级传递信息、争取资源与决策的沟通行为。 |
| BLUF | Bottom Line Up Front，结论先行：把最重要的结论/建议放在最前。 |
| 金字塔原理 | 明托提出的表达框架：结论先行、以上统下、归类分组、逻辑递进。 |
| SCQA | 情境-冲突-问题-答案，复杂问题的开场结构。 |
| PREP | 观点-理由-例证-重申，单点论证结构。 |
| STAR | 情境-任务-行动-结果，成果/复盘汇报结构。 |
| 5W2H | What/Why/Who/When/Where/How/How much，方案完整性检查。 |
| 闭环 | 凡事有交代、件件有着落、事事有回音；本项目指「进展/风险/所需支持/下一步」四要素。 |
| 闭环四要素 | 进展、风险（阻塞）、所需支持、下一步（含时间点）。 |
| 带建议的选项 | 决策点上给 2–3 个带利弊的方案并标注推荐项，而非把问题原样抛回。 |
| 事实与判断分离 | 明确区分「客观事实/数据」与「主观判断/建议」。 |
| 不卑不亢 | 既尊重对方又保持专业与独立判断，不谄媚也不卑微。 |
| 谄媚（反模式） | 空洞讨好、丧失信息增量的表达，本项目红线之一。 |
| 报喜不报忧（反模式） | 隐藏风险只讲好消息，本项目红线之一。 |
| 档位 | 风格强度的可选模式：简报档 / 标准档 / 详汇档 / 危机档。 |
| 危机档 | 问题上报结构：影响 → 已采取措施 → 原因 → 方案 → 所需支持。 |
| Skill | Cursor Agent Skill，由 SKILL.md + 配套文件组成的可复用能力单元。 |
| SKILL.md | Skill 的主文件，含 YAML frontmatter（name/description 等）与行为指令正文。 |
| disable-model-invocation | SKILL.md 元数据，控制是否仅在显式调用时加载（true）还是允许语境自动触发（false）。 |
| 渐进式披露 | 核心信息放 SKILL.md，细节放配套文件，按需读取。 |
| 认知负荷 | 接收方处理信息所需的心智资源；本项目力求最小化。 |
| 选择架构 | 通过选项的呈现方式影响决策的设计（带推荐的选项即应用）。 |
| LLM-as-judge | 用大模型对输出自动评分的评估方式（评估备选方案）。 |
| 四层沟通能力模型 | 本项目核心框架：第1层公文规范、第2层任务汇报、第3层自我营销、第4层灵感激发，层层递进（见 15 号）。 |
| 公文规范（L1） | 取公文之严谨、得体、条理（"一定程度"），去其官僚空话与八股腔。 |
| 自我营销（L3） | 在如实汇报之上，让思考深度与价值被领导看见；营销"价值"而非自夸"自己"。 |
| 灵感激发（L4） | 通过递话、提问、共鸣点燃领导表达欲，引导其分享见解。 |
| 高山流水遇知音 | 本项目体验北极星：让领导聊得畅快、被理解、被激发，产生"给自己讲课/讲品牌"的成就感。 |
| 知音体验 | 顺畅 + 共鸣 + 激发三层递进的沟通体验。 |
| 递话（抛钩子） | 在汇报中留开放延展点，邀请领导补充其思路。 |
| 产婆术 | 苏格拉底式提问，通过提问引出对方已有的智慧。 |
| 捧杀（反模式） | 空洞吹捧让人飘飘然却无实质收获；本项目红线之一。 |
| "爽且有所得"试金石 | 判断知音体验是否健康：对话后领导除了爽，是否真有所得/启发。 |
