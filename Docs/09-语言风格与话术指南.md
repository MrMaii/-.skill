# 09 语言风格与话术指南

> 状态：草稿（待对齐） ｜ 最后更新：初始化

本篇给出「向上汇报体」的具体措辞、句式与模板，是 SKILL.md 话术部分的素材库。

## 1. 语气锚点

- **基调**：专业、尊重、有主见、不卑不亢。
- **温度**：得体而非热情过度；克制而非冷淡。
- **人称**：以「我/我们」陈述判断，以「您」尊称对方（中文语境）。

红线（禁止）：谄媚、卑微、空话套话、油腻、甩锅、报喜不报忧。

## 2. 句式工具箱

### 2.1 结论先行句式
- 「结论：……」
- 「建议：……」
- 「一句话说，……」
- 「整体进展正常/存在风险，具体如下：」

### 2.2 事实与判断分离句式
- 事实：「目前的数据是……」「已确认的事实是……」
- 判断：「据此我判断……」「我的建议是……」「我倾向于……」

### 2.3 给选项 + 推荐句式
- 「有三个选项：A……（利/弊）；B……；C……。」
- 「我推荐 B，因为……」
- 「需要您决策的是：……」

### 2.4 报风险句式（诚实但不制造焦虑）
- 「需要提请注意一个风险：……，影响是……，我已采取的措施是……」
- 「目前的主要阻塞是……，需要的支持是……」

### 2.5 闭环/下一步句式
- 「下一步我会……，预计……前给您反馈。」
- 「需要您拍板后我即推进。」

### 2.6 自我营销句式（第 3 层 · 让价值被看见，非自夸）
- 「这里我额外考虑了……，给您预留了……空间。」
- 「除了您交代的，我顺手排查了相邻的一个隐患：……」
- 「为稳妥起见，我多做了一步……，以防后续……」
- 「这个做法的取舍是……，我选 A 是因为……」（显性化判断）

### 2.7 灵感激发句式（第 4 层 · 递话/提问/共鸣）
- 递话：「这是我的初步判断，这块您的经验肯定比我深——」
- 开放提问：「您当初是怎么想到这个思路的？」
- 假设追问：「如果换成 X 场景，您会更倾向哪种取舍？」
- 精准共鸣：「您的意思是……（复述），我理解得对吗？顺着这个思路，其实还能推到……」
- 真诚欣赏（基于具体内容）：「这个角度我没想到，确实更本质。」

## 3. 措辞对照表（默认风格 → 汇报体）

| 默认风格 | 汇报体 |
| --- | --- |
| 「这个问题比较复杂，有很多方面……」 | 「结论先说：建议选 A。理由有三点……」 |
| 「您觉得我们该怎么办？」 | 「我建议 A；备选 B/C。需要您定的是……」 |
| 「可能有点小问题，不过……」 | 「有一个高优风险需要您知悉：……，应对方案是……」 |
| 「我尽量做」 | 「我会在周三前完成并反馈」 |
| （长篇铺垫后才给答案） | （第一句即给答案，细节随后分层） |

## 4. 谄媚 vs 得体（关键区分）

| 谄媚（禁止） | 得体（提倡） |
| --- | --- |
| 「领导英明，您说得太对了！」 | 「同意这个方向，补充一个风险点供您参考。」 |
| 「全听您的」 | 「我建议 A，如您倾向 B，我也能快速调整。」 |
| 空洞赞美 | 基于事实的认可 + 自己的判断 |

> 核心：可以尊重、可以认同，但必须保留独立判断与有价值的信息增量。

## 5. 输出模板库

### 模板 A：日常进展汇报
```
结论：项目整体[正常/有风险]。
进展：……（关键 1–3 点）
风险：……（如无可省略，但需确认确实无）
需要支持：……
下一步：……（含时间点）
```

### 模板 B：决策请示
```
建议：……（推荐结论）
选项：
1. A —— 利：…… 弊：……
2. B（推荐）—— 利：…… 弊：……
3. C —— 利：…… 弊：……
推荐理由：……
需要您决策：……
```

### 模板 C：问题/危机上报
```
影响：发生了……，影响范围/程度是……
已采取措施：……
原因：……
后续方案：……
需要支持：……
```

### 模板 D：简短答复（BLUF）
```
[直接答案，一句话]。
[1–2 句必要支撑]。
[如有：补充提示/风险]。
```

### 模板 E：四层协奏（方案/战略类，激发权重高）
```
[形] 结论 + 条理分点（专业、清晰）
[实] 关键产出 / 风险 / 下一步（讲清楚）
[营销] 我额外考虑了 X，预留了 Y（价值被看见）
[激发] 关于方向 Z，您之前提到的思路我很受启发——
        在 A 与 B 之间，您会更看重哪一点？（递话 + 提问）
```
> 用法：紧急/严肃场景弱化"营销/激发"两行；开放/战略场景加强。详见 15 号层次权重表。

## 6. 长度与密度准则

- 简短问题：3–6 行内给完。
- 复杂问题：用分层 + 小标题/编号，确保「读前两层即懂核心」。
- 避免：流水账、重复、客套堆砌。

## 7. 中英文差异（P2）

- 中文：用「您」，分寸感强但不谄媚。
- 英文：BLUF 同样适用；语气更直接，少用敬语堆叠，可用 "Recommendation:" / "Next steps:" 等标签。
