# 11 开源策略与协作

> 状态：草稿（待对齐） ｜ 最后更新：初始化

本篇规划项目作为开源项目的许可、仓库结构、贡献流程与社区运营。

## 1. 开源目标

- 让任何人可以**免费使用、复制、改造**这个汇报体 Skill。
- 成为「沟通风格类 Skill」的一个高质量、可学习的范本。
- 通过社区反馈持续迭代话术与评估用例。

## 2. 许可证（已确认：MIT）

**项目采用 MIT 许可证**（已与项目方确认）：最宽松，可自由使用、修改、商用与再分发，最大化社区采用。

- 代码 / Skill 文件：MIT。
- `Docs/` 文档：默认随仓库一并适用 MIT（如后续希望文档单独用 CC BY 可再议）。
- 待办：在仓库根目录添加 `LICENSE` 文件（标准 MIT 文本 + 版权署名）。

## 3. 仓库结构（草案）

```
report-style-skill/
├── README.md                 # 项目介绍、快速上手、效果示例
├── LICENSE                   # 许可证
├── CONTRIBUTING.md           # 贡献指南
├── CODE_OF_CONDUCT.md        # 行为准则
├── CHANGELOG.md              # 版本变更
├── skill/                    # 可直接使用的 Skill 产物
│   ├── SKILL.md
│   ├── reference.md
│   ├── examples.md
│   └── anti-patterns.md
├── Docs/                     # 设计与分析文档（本目录）
└── tests/                    # 评估用例（见 13 号）
```

> 说明：`Docs/` 解释「为什么」，`skill/` 是「拿来即用」的成品，`tests/` 保证质量。

## 4. README 要点（草案）

- 一句话定位 + 一张「前后对比」效果图/示例；
- 30 秒快速上手（如何安装到 `~/.cursor/skills/`）；
- 适用 / 不适用场景；
- 示例输出；
- 如何贡献；
- 许可证。

## 5. 贡献流程（草案）

1. Issue 先行：提想法/报问题。
2. 改话术/规则：附上「改前 vs 改后」示例。
3. 改行为：需补充/更新对应测试用例（13 号）。
4. PR 模板：变更说明 + 影响范围 + 自测结果。
5. 评审标准：是否符合核心原则、是否触碰红线/反模式。

## 6. 质量门禁

- 任何对核心行为的修改，必须通过评估用例（13 号）。
- 新增话术需避开反模式（10 号）。
- 保持术语一致（14 号）。

## 7. 社区运营（轻量）

- 收集真实使用反馈，沉淀为新的场景/用例。
- 维护「优秀汇报示例」与「踩坑案例」两个集合。
- 长期：孵化「沟通风格 Skill 家族」。
