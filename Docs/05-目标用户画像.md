# 05 目标用户画像

> 状态：草稿（待对齐） ｜ 最后更新：初始化

> 重要前提（见 01 号 Decision Log）：本 Skill 面向 **AI 开发者 / AI 使用者**。在产品里，**AI 永远是「下属」，使用者永远是「领导」**。本文档的画像指的是「会安装/使用这个 Skill 的人」，而不是用户的现实职场身份。

## 1. 用户分层概览

| 角色 | 与本 Skill 的关系 | 主要诉求 |
| --- | --- | --- |
| AI 重度使用者 | 安装使用者 | AI 干完活的汇报能一眼看懂、抓住重点 |
| AI 开发者 / Agent 构建者 | 集成者 | 让自家 AI 的汇报更专业、更可信 |
| 团队技术负责人 | 推广者 | 统一团队 AI 助手的汇报风格 |
| 开源社区 | 复用/改造者 | 一个可学习的风格 Skill 范本 |

## 2. 核心画像（Personas）

### Persona A：小航 · AI 重度使用者
- **背景**：每天用 Cursor / AI 助手处理大量任务（写代码、查资料、做分析）。
- **痛点**：AI 干完活的汇报是大段流水账，要自己读半天才知道「做了啥、有没有坑、接下来干嘛」。
- **期望**：AI 汇报时结论先行、风险清晰、下一步明确，像个靠谱下属。
- **成功画面**：扫一眼汇报就知道结果与待办，不用逐行读。

### Persona B：Leo · AI 开发者 / Agent 构建者
- **背景**：在做自己的 AI 产品 / Agent，关心输出质量与可信度。
- **痛点**：默认输出语气平、缺重点，显得不够专业。
- **期望**：挂上一个 Skill 就能让 AI 的汇报变「职业范」。
- **成功画面**：集成后，产品的汇报体验明显更专业、可信。

### Persona C：Mia · 团队技术负责人（推广者）
- **背景**：希望团队内 AI 助手输出风格统一、专业。
- **痛点**：不同人用 AI 风格不一，汇报质量参差。
- **期望**：一个可共享、可版本化的 Skill，团队统一启用。
- **成功画面**：团队 AI 汇报风格一致且高质量。

## 3. 用户需求优先级（KANO 视角）

| 需求 | 类型 | 说明 |
| --- | --- | --- |
| 结论先行 | 基本型 | 没有会很失望 |
| 风险/闭环要素 | 期望型 | 越完整越满意 |
| 带建议的选项 | 兴奋型 | 超出预期，强差异化 |
| 不卑不亢的语气 | 基本型 | 谄媚/卑微会直接劝退 |
| 可关闭/调档 | 期望型 | 控制感 |

## 4. 反向画像（谁不是目标用户 / 何时不该用）

- 需要**轻松闲聊**、情感陪伴的场景——汇报体会显得生硬。
- 需要**头脑风暴发散**、自由联想的场景——汇报体偏收敛。
- 纯创意写作、文学创作——风格不匹配。

> 详见 10 号《反模式与使用边界》。
