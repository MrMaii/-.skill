# 12 产品路线图

> 状态：草稿（待对齐） ｜ 最后更新：初始化

## 1. 版本规划总览

| 版本 | 主题 | 核心交付 |
| --- | --- | --- |
| v0.1 (MVP) | 能用的汇报体 | SKILL.md + 核心话术 + 基础用例 |
| v0.2 | 场景完善 | 覆盖主要场景 + 反模式护栏 |
| v0.3 | 可调控 | 档位机制 + 触发/关闭 |
| v0.5 | 开源就绪 | README/LICENSE/贡献流程 + 测试集 |
| v1.0 | 稳定发布 | 评估达标 + 文档完善 + 示例丰富 |
| v1.x+ | 扩展 | 多语言、风格家族 |

## 2. 里程碑详述

### M1 — MVP（v0.1）
- [ ] 确定命名、定位、默认触发方式
- [ ] 编写 `SKILL.md`（核心 7 原则 + BLUF/决策/危机模板）
- [ ] 3 个核心场景：进展汇报、决策请示、问题上报
- [ ] 10 条基础测试用例
- **验收**：核心场景输出稳定呈现结论先行 + 闭环。

### M2 — 场景完善 + 高层能力（v0.2）
- [ ] 补齐复盘(STAR)、方案(5W2H)、对上答复场景
- [ ] 落地第 3 层「自我营销」与第 4 层「灵感激发」话术
- [ ] 完成高层能力的反模式护栏（谄媚/捧杀/操纵边界）
- [ ] 扩充话术库与对照表
- **验收**：四层能力可见、无谄媚/捧杀样本。

### M3 — 可调控（v0.3）
- [ ] 档位机制（简报/标准/详汇/危机）
- [ ] 触发、降级、关闭逻辑
- **验收**：用户可一句话切换/关闭风格。

### M4 — 开源就绪（v0.5）
- [ ] 仓库结构、README、LICENSE、CONTRIBUTING
- [ ] 测试集成体系（13 号）
- [ ] 前后对比示例
- **验收**：外部用户可独立安装使用。

### M5 — 稳定发布（v1.0）
- [ ] 评估通过率达阈值
- [ ] 盲测偏好率达标
- [ ] 文档与示例完善
- **验收**：达到对外正式发布标准。

### M6+ — 扩展（v1.x）
- [ ] 多语言（中/英）
- [ ] 沟通风格 Skill 家族（对外谈判体、客服体等）
- [ ] 与更多平台/Agent 的集成示例

## 3. 优先级原则

1. 先把「向上汇报」单一方向做扎实，再扩展变体。
2. 先质量（评估）后规模（多场景/多语言）。
3. 先显式触发求稳，再放开自动触发。

## 4. 风险与依赖

- 依赖底层模型能力，效果存在波动 → 用评估用例兜底。
- 风格主观，验收阈值需共识 → 早定评估标准。
