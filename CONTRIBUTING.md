# 贡献指南（Contributing）

感谢你愿意一起把「向上汇报体」打磨得更好！

## 怎么开始

1. 先开一个 Issue 聊聊你的想法或要修的问题。
2. Fork 仓库，基于 `main` 建分支。
3. 提交 PR，关联对应 Issue。

## 贡献类型与要求

| 你想改 | 请附上 |
| --- | --- |
| 话术 / 风格（`skill/` 或 `.cursor/skills/`） | "改前 vs 改后"对比示例 |
| 行为规则（SKILL.md） | 补充 / 更新对应的[测试用例](tests/cases.md) |
| 设计文档（`Docs/`） | 保持术语一致（见 [Docs/14 术语表](Docs/14-术语表.md)） |
| Bug / 体验问题 | 复现步骤或截图 |

## 质量门禁（PR 合并前）

- [ ] 自测用例通过、红线零违规（见 [tests/self-test-report.md](tests/self-test-report.md)）。
- [ ] 没有引入谄媚 / 捧杀 / 编造数据等反模式（见 skill 的 `anti-patterns.md`）。
- [ ] 命名与术语全仓一致。
- [ ] 文档内部链接可达。

## 行为准则

参与本项目即表示你同意遵守友善、尊重、对事不对人的协作原则。

## 许可证

提交贡献即表示你同意以 [MIT 许可证](LICENSE) 授权你的贡献。
