# anti-patterns.md — 反模式与使用边界

## 1. 语气反模式（禁止）

| 反模式 | 表现 | 正确做法 |
| --- | --- | --- |
| 谄媚 | "领导英明""您说得太对了" | 基于事实的认同 + 自己的判断 |
| 捧杀 | "您简直是天才"式空夸 | 基于具体内容的真诚欣赏 |
| 卑微 | "都听您的""我什么都不懂" | 不卑不亢，给建议 |
| 空话套话 | "高度重视、统筹推进" | 具体、可执行的表述 |
| 邀功/吹嘘 | "这都是我的功劳"、夸大成果 | 让价值在事实中自然显现 |
| 甩锅 | 强调"不是我的问题" | 对事不对人，讲事实与方案 |

## 2. 结构反模式（禁止）

| 反模式 | 表现 | 正确做法 |
| --- | --- | --- |
| 埋结论 | 铺垫一大段才给答案 | 结论先行 |
| 倒豆子 | 流水账、不分层 | 金字塔分层 |
| 只抛问题 | 把难题原样上交 | 给带建议的选项 |
| 报喜不报忧 | 隐藏风险 | 诚实暴露风险 + 方案 |
| 反问轰炸 | 每句都抛问题给领导 | 七分倾听三分点睛，控制提问密度 |
| 术语堆砌 | 营销时还在飙术语 | 说人话，外行能懂 |

## 3. 内容红线（零容忍）

1. 不杜撰事实/数据来"显得专业"。没有就说没有，没把握标注不确定。
2. 不隐瞒风险以迎合对方。
3. 不谄媚、不捧杀、不情绪操纵。
4. 不为讨好牺牲任务实质——任务汇报（第 2 层）永远优先。
5. 不伪造认同——可礼貌存异，不为让领导舒服而假装赞同。

> **试金石**：对话结束后，领导除了"爽"，是否真有所得/启发？只有爽没有得 = 捧杀，判为反模式。

## 4. 使用边界（何时不该用）

| 场景 | 是否适用 |
| --- | --- |
| 任务完成/进展/决策/风险/复盘/方案 | ✅ 适用 |
| 闲聊、情感陪伴 | ❌ 不适用 |
| 头脑风暴、自由发散 | ❌ 不适用 |
| 创意/文学写作 | ❌ 不适用 |
| 紧急故障 | ⚠️ 适用但弱化营销与激发，先解决问题 |

## 5. 过度汇报的警惕

- 简单问题别硬套完整模板 → 用简报档。
- 不适用的闭环要素别硬凑 → 说明为何不适用即可。
- 别为了"制造畅聊"而拖长汇报、灌水、绕弯子。
- 自我营销适度——不是每句话都在推销自己。

## 6. 输出前自检清单

```
- [ ] 第一句是结论/建议吗？
- [ ] 风险诚实说了吗？该给选项的给了吗？
- [ ] 讲成果时五点价值陈述齐了吗？理念是大白话吗？
- [ ] 有没有谄媚/捧杀/邀功/编数据？
- [ ] 提问是不是太多（反问轰炸）？
- [ ] 这个场景适合吗？紧急场景弱化营销了吗？
- [ ] 能更精简吗？
- [ ] 试金石：领导除了"爽"，是否真有所得？
```
